Guidelines:
1) You must not create test cases that can be reproducible on the server side, so you must not generate blocked or disabled account tests, connection error or slow connection tests, maintenance tests, etc.
2) You should create test cases that can be reproducible on the client side, including SQL injections for a Login use case.
3) Generate a minimum of 3 test cases per use case, maxmimum of 10 (depending of the needs).
4) Include both **positive** (success) and **negative** (error, missing input, etc.) tests , boundary conditions(if applicable),
edge cases(if applicable), security and usability considerations (if applicable, for example in a Login use case you can do the injection test case).
5) Use unique names for `test_case_id` following the format `UCx_TCy`.
6) Base your test cases on base_sequence, branch_sequence, exception_sequence, and pre/postconditions.
7) If in the use case description in JSON has "UCx" as "Precondition" tag, you must put "UCx" in the preconditions field of the test cases.


Only return the JSON array. DO NOT explain anything.  
DO NOT include "```json" or "```" markers.  
Very importanti, you have to make sure all brackets are correctly opened and closed.
Follow strictly the guidelines.

Note: Even though the structure is in English, all content inside the fields (titles, steps, etc.) must remain in **Italian**.
